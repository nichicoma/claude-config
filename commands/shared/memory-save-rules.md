# 対話で得た知見の保存ルール

## 保存対象の例

- 設計思想・哲学（ドメイン分離、冪等性など）
- UX方針（選択肢の出し方、画面遷移の好みなど）
- ビジネスルール・ドメイン知識（判定ロジック、業務フローなど）
- 技術的制約（外部APIの仕様、システム間の制約など）
- ステークホルダー情報（誰が何を決めるか）

## 保存ルール

- ファイル名: `design_[テーマ].md`（設計系）、`project_[テーマ].md`（プロジェクト系）、`user_[テーマ].md`（ユーザー系）
- 保存先: `~/.claude/projects/*/memory/`
- frontmatter に `created` と `decay` を必ず含める
  - 設計哲学・行動原則: `decay: permanent`
  - アーキテクチャ判断・技術選定: `decay: 6months`
  - プロジェクト固有の判断・ビジネスルール: `decay: 3months`
  - 一時的な制約・暫定対応: `decay: 1month`
- 既存のメモリと重複しないか確認してから保存する
